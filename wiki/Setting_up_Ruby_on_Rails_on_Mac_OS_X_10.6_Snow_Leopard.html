Setting up Ruby on Rails on Mac OS X 10.6 Snow LeopardEdit

Created 2009-08-31, updated 2017-05-18

Note: These build notes were made while working with a first-generation Intel-powered iMac, and as such are specific to a 32-bit architecture. For notes made building on a 64-bit platform see "Setting up Ruby on Rails on Max OS X 10.6.2 Snow Leopard".

Mac OS X already comes with the basic required pieces:

Other requirements

My workflow requires nginx, memcached and MySQL. In addition to installing these on my local Snow Leopard machine I took the opportunity to update a couple of components on the server (nginx and memcached) so as to have exactly the same versions on both my deployment and development machines.

nginx

See:

  • Installing nginx 0.7.61 on Mac OS X 10.6 Snow Leopard
  • Updating to nginx 0.7.61 on Red Hat Enterprise Linux 5.3

memcached

See:

  • Installing memcached 1.4.1 on Mac OS X 10.6 Snow Leopard
  • Updating to memcached 1.4.1 on Red Hat Enterprise Linux 5.3

MySQL

See:

  • Installing MySQL 5.0.85 on Mac OS X 10.6 Snow Leopard

Note that this is not an exact match with the version running on the server (5.0.45-7.el5), but it is at least in the same major version series. I did not update to the 5.1 "GA" (General Availability) release on the server because I did not want to lose the automatic patches pushed out via the Red Hat network.

Other prerequisites

We'll need the mysql gem, which is considerably easier to build on Snow Leopard than it was on Leopard (see "Installing the mysql gem on Mac OS X Leopard 10.5.1":

$ sudo gem install mysql

And Rack. For some reason, Rails complains even when it has its own built-in version of Rack:

$ script/server
/System/Library/Frameworks/Ruby.framework/Versions/1.8/usr/lib/ruby/1.8/rubygems.rb:636:in `report_activate_error': Could not find RubyGem rack (~> 1.0.0) (Gem::LoadError)

So we install it:

$ sudo gem install rack

And now it works:

$ script/server
=> Booting Mongrel
=> Rails 2.3.3 application starting on http://0.0.0.0:3000
=> Call with -d to detach
=> Ctrl-C to shutdown server

Note that to actually do anything useful with the server you'll need to re-create your databases or migrate them. See "Migrating MySQL databases" for an example.
