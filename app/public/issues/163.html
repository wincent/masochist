Bug #163: Update Floater when Internet radio streaming server changes tracks

Kind bug
Product Synergy
When Created 2005-03-27T22:53:36Z, updated 2006-07-05T02:04:57Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"Synergy rocks! I'm one of the few? (many?) streaming radio listeners, so if Synergy could automatically display the track info banner everytime the streaming server changes tracks, that would be an awesome new feature!"

My reply:

This is a current shortcoming of Synergy. If using a recent version of Synergy with iTunes 4.7 or later then Synergy *does* receive notification of title changes, but it doesn't reflect the changes in the Floater because it doesn't realize that the track has changed (the id numbers used to identify the "track" are still the same).

Synergy Advance has a completely different communications subsystem which doesn't have this flaw. I recommend trying it out when it comes out (the first public preview release is due out next month).

In the meantime I will leave this bug open in the tracking database so that the issue doesn't get forgotten for Synergy Classic. It would be quite tricky to re-work the current implementation to eliminate the flaw, but it's not impossible.
