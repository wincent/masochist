Bug #1784: Chokes on umlauts and other non-ASCII chars in URLs in external links

Kind bug
Product wikitext
When 2011-02-17T14:01:56Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

Just noticed that we're choking on:

[http://chrishecker.com/Kurt_Gödel_is_Laughing_His_Ass_Off_Right_Now here]

As a work around, can encode the non-ASCII in the URL using percent encoding (ie. http://chrishecker.com/Kurt_G%f6del_is_Laughing_His_Ass_Off_Right_Now); note how this works.
