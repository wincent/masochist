Bug #545: CRASH: Crash in initializedCachedState -> iconImageWithBadge:trackProgress:

Kind bug
Product Synergy Advance
When Created 2007-02-21T08:22:34Z, updated 2007-05-15T12:20:57Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

In response to a user-submitted crash report (will attach the report).

This crash is occurring in the "iconImageWithBadge:trackProgress:" method (called from "initializeCachedState"). All of the code for managing the Dock icon has been totally rewritten in order to take advantage of Tiger's Core Image API, and the methods in this stack trace don't even exist anymore, so I don't believe this problem (whatever the cause) exists any more. I am going to add the fixed-in-prerelease keyword.

Still, I am curious as to what kind of exception was raised. Was anything printed to the console at the time?

Here are the methods implicated in the crash.

0xe1001b9c -[NSImage(SynergyAdvance) iconImageWithBadge:trackProgress:] (in SynergyAdvance_ppc_unstripped) 0xe1007678 -[WOSynergyAdvanceFrameworkClass04500 setBadge:] (in 0x0000e918 -[WOSynergyAdvanceController initializeCachedState] (in Synergy Advance_ppc_unstripped) 0x00009e18 -[WOSynergyAdvanceController awakeFromNib] (in Synergy Advance_ppc_unstripped)
