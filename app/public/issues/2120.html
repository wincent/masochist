Bug #2120: Hotkey Ctrl-/ can't be registered, doesn't fire (under Mavericks?)

Kind bug
Product Synergy
When Created 2013-11-21T16:54:07Z, updated 2013-12-28T22:10:44Z
Status open
Reporter beporter
Tags no tags

Description

I just upgraded from ML to Mavericks last night. Today, my Synergy hotkey for "Play/Pause" no longer has any effect on iTunes when pressed. Additionally, there seems to be a class of Ctrl-_ combinations that I can no longer *assign* as hotkeys in Synergy Preferences.

For example:

  • Ctrl-/ does not work.
  • Ctrl-Opt-/ does not work.
  • Ctrl-Shift-? works.

Comments

  1. beporter 2013-11-21T19:45:48Z

    I take some of this back-- after messing with the new System Preferences > Security & Privacy > Accessibility panel and adding Synergy Preferences (and the bundled Synergy.app) in there, I am able to get the hotkeys working again.

    However, the caveat is that Synergy Preferences no longer seems to be able to properly stop a running background instance of Synergy.app (requiring a manual kill -9 from the command line or force quit in Activity Monitor). I haven't determined all of the cases where this is true, however just clicking "Apply" in Synergy Preferences after changing hotkeys no longer updates the bound hotkeys in realtime-- it's necessary to kill the background Synergy.app and relaunch it for the keys to work again.
