Feature request #171: REQUEST: Control over which events trigger "Track Change Items"

Kind feature request
Product Synergy Advance
When Created 2005-04-11T18:40:16Z, updated 2006-07-05T02:04:58Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From this forum post:

"I do have one more request - it would be nice if there was a way to toggle whether or not Synergy actually runs the scripts or not. I can see times when I wouldn't want the music I'm listening to appear as my iChat status, for example, and right now the only way to deactivate this feature is to physically move the script out of the Track Change Items folder.

Having this tied to a quick key and also in the Synergy menubar menu would be perfect - could this be added? "

http://wincent.com/a/support/forums/showflat.php?Number=577

It's fairly obvious from this thread that people need control not only over which Track Change Items get triggered (see bug #169) but also when they get triggered. This bug will be used to track the feature request about the latter. It should be possible to use checkboxes to control whether a Track Change Item gets run on player state changes, track info changes, or both.

This feature will share a preference pane (most likely to be called "Scripts" or "Scripting") with the features requested in bug #170, bug #171 etc). I may also be able to add a "Activate/Deactivate Track Change Items" toggle in the menu but I am not yet sure how it should be labelled.
