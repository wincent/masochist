Bug #235: Pre-select "Current user" or "All users" based on contents of disk

Kind bug
Product Install
When Created 2005-05-13T23:34:15Z, updated 2006-07-05T02:05:37Z
Status open
Reporter John Burgess
Tags no tags

Description

WHen upgrading syneregy (say from1.8 to1.9), I don't remember whether I installed it for me only or for everyone on the system. From previoius experience, I recall that it installs things in different places and I end up with two sets of files from different versions. Please have the installer check for an existing install, figure out which type it was, and default to that for the current upgrade/install.

Thanks!

Comments

  1. Greg Hurrell 2005-05-14T00:28:41Z

    Thanks for the suggestion. It's a good one. Synergy 1.9 uses Install 1.3 and so doesn't suffer from the problem you talk about.

    Basically, if you try to install for "current user only" and there are still old files for "all users" littered around the system it will get rid of the old files (the only drawback is it may ask you for an administrator password if it doesn't have privileges to remove those files).

    The reverse case (old "current user only" files being left around when you install for "all users") has never been a problem because the "current user only" files have always been automatically removed when installing for "all users".

    So the problem you mention here (having two sets of files which come from different versions) is no longer possible.

    Nevertheless I think your suggestion is a good one. I have reworded the summary slightly to reflect the sort of change that would be needed: basically, it would be nice if Install could detect if there were any "all users" files on the disk when you run the installer, and if there are, automatically pre-select the "all users" radio button. In the reverse case (no "all users" files on disk) it could either select the "current user" radio button if it finds "current user" files on the disk, or leave the selection untouched.
