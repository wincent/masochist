Feature request #1858: Reverse the direction of results, starting them at bottom instead of top

Kind feature request
Product Command-T
When 2011-08-25T16:37:39Z
Status closed
Reporter anonymous
Tags no tags

Description

When I'm using Command-T in Textmate, and I type the file name/path I'm looking for, I don't have to finish typing it because my eyes can keep looking in that same exact spot (the top row of the table) the whole time, because the most-likely-result position is always in the same location on my screen. However, in this plugin, the height of the table varies, and so the location of the most-likely-result jumps across my screen as I'm typing. This makes it harder to use, and I usually just type the whole filename anyway, instead of just typing what I need. If the list were reversed, the position of the most-likely-result will stay at the bottom of the list at all times, so no matter what height the table is or becomes, my eyes can fixate on it easily. Even if this is simply an option to enable but which is disabled by default, it would be largely beneficial to me and probably many others.
