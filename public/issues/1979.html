Feature request #1979: Open specific file types using native OS handler

Kind feature request
Product Command-T
When 2012-05-22T15:46:12Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

Btw, it'll be great if Command-T can also provide an option to open the selected file with the default native viewer of the OS. For example, if a PDF file is selected, hitting, e.g. CTRL-ENTER, will open the file with the default PDF viewer on that OS, e.g. Adobe Reader. Similarly, if ODT file is selected, it'll open that file with LibreOffice Writer.

In Linux X Windows, the command "xdg-open <*/path/filename*>" opens a file using its default program. The "start" command does the same thing in Windows. If Command-T can provide an option to pass the selected path/filename to user's customised external program, that'll be great.
