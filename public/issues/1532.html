Feature request #1532: Unified login/sign-up form

Kind feature request
Product wincent.com
When Created 2010-04-04T05:10:38Z, updated 2010-04-18T16:03:38Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

Still need to hammer out the details, but what we currently have is:

  • Login/Logout link in top right of navbar
    • Login link takes you to: https://wincent.com/login
      • Login page has form with "Email address" and "Passphrase" fields, a "Log in" button, and links to "Sign up" (https://wincent.com/users/new) and "Forgot your passphrase?" (https://wincent.com/resets/new)

While this isn't bad we could probably streamline it:

  • Possibly rename "Login"/"Logout" link to "Account" or "My Account" or something similar; not sure about this one
  • Clicking on Login/Logout link drops down an overlaid tab loaded via AJAX from https://wincent.com/login (browsers without JavaScript just load that page directly instead)
    • Same fields ("Email address" and "Passphrase"), but now the "Log in" button reads "Log in or sign up"
    • Alternatively, could use the Amazon/Google approach and add a radio button with two choices — "I already have an account" and "I'd like to create a new account", or similar &mash; which updates the text of the button to "Log in" or "Sign up" according to which radio button is checked

There are lots of articles out there on this subject but here are some good ones.

Usability studies

  • http://sites.google.com/site/oauthgoog/UXFedLogin: "Usability Research on Federated Login"
  • http://blog.leahculver.com/2009/11/log-in-or-sign-up.html#comments: "Log in or sign up?"

Implementation examples

  • http://aext.net/2009/08/perfect-sign-in-dropdown-box-likes-twitter-with-jquery/: "Perfect signin dropdown box likes Twitter with jQuery"
