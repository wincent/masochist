Feature request #1279: Set up "l" or "s" as shortcut path for "links" controller

Kind feature request
Product wincent.com
When Created 2009-04-17T11:33:48Z, updated 2009-04-24T06:27:57Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

One of the purposes of the "links" controller is to enable the creation of convenient shortcut links that redirect to hard-to-remember or cumbersome-to-type URLs both inside and outside the site.

For example, just say we had an wiki article called "How to file a good bug report", it would be painful to have to type "see [[How to file a good bug report]]" all the time when one could instead type "see [/links/bugs this]".

To make it even more convenient to use the short links, it would be nice to set up "l" as an even shorter route to the controller, so the above example would be just "see [/l/bugs this]".

Admittedly, when you're writing entries within the app it's not that hard to do the full links, but when you're not in the app but you still want to refer to specific pages (for example, when you're writing an email) it could be a real godsend to be able to write stuff like "see https://wincent.com/l/bugs" instead of "see https://wincent.com/wiki/How_to_file_a_good_bug_report".
