Bug #1562: Up & Down arrow keys require pressing and releasing the arrow key as opposed to simply holding the arrow key to raise or lower the volume.

Kind bug
Product Synergy
When Created 2010-06-08T21:43:57Z, updated 2010-06-09T10:48:29Z
Status closed
Reporter anonymous
Tags no tags

Description

Hello there,

I've noticed that it is now necessary for me to press and release the arrow key while holding down the command key), as opposed to the previous method, which allowed me to press the command key, and then simply hold the up or down arrow, which would rapidly raise or lower the volume.

I've looked in Synergy's preferences pane, but i can't seem to find an option for this feature.

I'm using a new MacBook Pro, running OS 10.6.3, along with synergy version 4.4.1.

Is there anything that I can do to restore this feature, as I really preferred this method?

Thanks very much for your help, and I look forward to hearing from you.

Sincerely, Bruce Mccombs

  1. Greg Hurrell 2010-06-09T10:48:21Z

    This is a duplicate of ticket #1549. Pasting in from that ticket:

    Indeed that is the way they used to work, but this had to be changed to work around a change that Apple made in Snow Leopard.

    This manifested itself as a "sticky" hot key problem (the volume keys would jam and stick when held), and although I tried to work around the problem, the only definitive way to prevent it was to remove the press-and-hold functionality all together.

    See ticket #1383, ticket #1417, ticket #1431 for examples of reports of "sticky" keys.

    Looking at the version history, I think it was version 4.2.1b (January 2010) which removed the press-and-hold functionality.

    If I can ever figure out a way to make things work the old way again, or if Apple makes a change that returns their API to the former behavior, then I'll certainly restore the press-and-hold behavior. But for now I don't think there's anything I can do. I tried pretty much everything I could before (see those tickets mentioned above) and nothing seemed to make any difference; Snow Leopard is simply dropping some key-release events on the floor.

    I don't know if this is a bug in the OS, perhaps a latent one, or something else, but I really don't know what I could possibly be doing to cause the system to randomly omit events.

    So will mark this ticket closed as a duplicate.
