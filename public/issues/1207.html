Feature request #1207: Display annotation comment immediately when changing issue metadata

Kind feature request
Product wincent.com
When 2009-01-25T13:58:09Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

Recently added metadata annotations for issues (see ticket #1198).

An admin user can change issue status from "Open" to "Closed", for example, and a comment will appear noting the change.

This works well, but it could work much nicer in the case of AJAX edits. As it currently stands the annotation is added, but you don't actually see it in the browser until you refresh the page.

So I would like to expand that AJAX action to do a little more; not only change the metadata, which in turn causes the comment to be created on the back end, but also fetch the comment and dynamically update the list of comments on the fly.
