Bug #535: WinSwitch transparent warning dialogs are empty (no visible text)

Kind bug
Product WinSwitch
When Created 2007-01-22T04:04:08Z, updated 2010-08-12T10:13:34Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

From a user email:

"a black rectangle that is somewhat translucent- but permanent- at least until I remove the custom icon."

ACTUAL BEHAVIOUR:

Will attach a screenshot. I have been able to confirm the problem here.

EXPECTED BEHAVIOUR:

The dialog should appear as shown here:

http://wincent.com/a/products/winswitch/screenshots/notification.jpg

CAUSE:

The cause of the problem is unknown but given that the problem is new it means that either Apple changed something in one of the Mac OS X updates that has come out since WinSwitch 3.2.1 was released causing this breakage, or this might be an Intel-only bug (will have to check with the user to see if he is running on Intel or not).

WORKAROUND:

The workaround for now is to go into the WinSwitch preferences and uncheck the last checkbox ("Show warning if Apple Fast User Switching menu added to menu bar") as shown here:

http://wincent.com/a/products/winswitch/screenshots/preferences.jpg

  2. Greg Hurrell 2010-08-12T10:13:12Z

    I'm marking all WinSwitch issues closed seeing as I personally no longer use it, and in fact haven't for around 4 years now.

    WinSwitch addressed a real problem with the initial implementation of Fast User Switching in Panther (released October 2003), namely, the excessive screen real estate that it chewed up. Apple fixed that problem in Tiger, if I recall correctly, which came out in April 2005 (or if I'm wrong about it being Tiger, then it was definitely fixed by the time Leopard came out, in October 2007).

    With this change, most of the justification for WinSwitch's existence went away, at least for me. So that's why I'm going to close all these tickets: I can't really support something that I don't use myself.

    But it's open source (see http://git.wincent.com/WinSwitch.git), so if any one wants to tackle any of these issues and submit patches, I'll be happy to accept them.
