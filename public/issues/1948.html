Bug #1948: ">> _" prompt is not always shown

Kind bug
Product Command-T
When Created 2012-02-13T02:18:09Z, updated 2012-02-15T14:24:17Z
Status open
Reporter Mario Grgic
Tags no tags

Description

I'm having this issue with Command-t 1.3.1 and VIM - Vi IMproved 7.3 (2010 Aug 15, compiled Aug 15 2010 22:03:01) on Mac OS X 10.6.8 and ruby 1.8.7 (2010-01-10 patchlevel 249) [universal-darwin10.0].

Basically when I invoke Command-T with <leader> T the prompt ">> " is not always shown and neither is the underscore cursor. Initially, the prompt is shown, but as soon as list of files is shown it disappears. If you start typing the prompt may appear again after you type more than 3 characters, but not always. Sometimes it will never show.

If I however hit Tab key and give focus to the match list, the prompt is shown immediately with what ever match text has been entered. If I then Tab back to the prompt the cursor is shown too.

Comments

  1. Mario Grgic 2012-02-15T14:24:17Z

    It turns out this was a VIM bug. After update to MacVim snapshot 62 the problem is gone. However, the performance is order of magnitude worse with large (over 50,000) number of files. I really hope this is not because of now more frequent prompt refreshes?
