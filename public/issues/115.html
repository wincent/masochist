Feature request #115: REQUEST: Please return polling to Synergy, so I can skip through songs silently

Kind feature request
Product Synergy
When Created 2005-02-04T06:25:36Z, updated 2006-07-05T02:04:17Z
Status open
Reporter bugzilla user 91
Tags no tags

Description

Before iTunes 4.7, Synergy polled iTunes to get info about what was playing. That no longer happens. Under the old model, I could jump from track to track using synergy, and the Floater would tell me what track was lined up to play, even if iTunes was stopped. Now, the Floater doesn't recognise that I have jumped to a new spot in the playlist, until the song starts playing. Synergy's main benefit for me was that it allowed iTunes to remain completely hidden (for me, behind Powerpoint 2004's Presenter Tools), and I could still control it. Now, I can't, because the Floater no longer tells me what track I've jumped to; I have to listen to the first couple seconds of the track to figure out if I'm where I thought I was. I request that Synergy automatically poll iTunes if a track jump has been performed, so the new status is reflected in the Floater. With iTunes 4.7 and Synergy 1.6.1, if I wish to jump to another song in a playlist, without having the sound appear, I would have to turn the sound off, make the jump with itunes playing so I can see where I land, hit the back button to return to the beginning of the song, hit pause, turn the sound on, then hit play again. This is quite different than jumping to a new song that I see immediately in the floater, and hitting play. Geoff

  1. Greg Hurrell 2005-02-05T08:06:59Z

    Thanks for submitting this as a request. There a few ways in which your problem might end up getting resolved:

    1. Synergy goes back to polling. I doubt this will happen, because most uses value the performance benefits of the new model too much.

    2. Synergy will do a single "poll" each time you skip tracks from within Synergy when iTunes is not playing. This won't completely deliver uniform behaviour, because if the user skips tracks in other ways (ie. the iTunes dock menu, clicking in iTunes itself etc), then no polling and therefore no updates will occur.

    3. Apple changes iTunes so that it posts notifications of track changes even when it's not playing. This is the ideal scenario, because it is both efficient and completely uniform. Here's the URL for iTunes feedback if you want to encourage Apple to make this change:

    http://www.apple.com/feedback/itunes.html

    My votes are for 3, 2, 1 in that order. Other users are welcome to leave comments here to help inform the decision about what gets done!
