  2. Greg Hurrell 2009-09-08T15:52:27Z

    No, there are no plans to update the old 3.4 series to address this issue.

    A total rewrite was done in the 3.5 series precisely to work around a compatibility problem between Garbage Collected applications and the System Preferences on Leopard. A that point Synergy stopped being a System Preferences panel and became a stand-alone app. So having done all that work to convert Synergy into a standalone application, I don't have any plans on going back to make changes to an older version.

    So your options here are moving to the 3.5 or 4.0 series, or putting up with the mode changes when you need to manipulate the preferences.

    As an aside, I'm not actually sure what would need to be done nor what's happening. Possibly because I am on one of the first Intel iMacs, which I guess always runs in 32-bit mode, so I've never seen this "change to 32-bit mode" that you mention although I have seen mentions of something that sound suspiciously like it on the forums.

    3.4 is a Universal Binary with PowerPC an Intel native code in it (32-bit both of them), so I guess what would be needed would be architecure, 64-bit Intel, as well. Would swell the binary size a bit, and I don't think I can actually test it on this machine.
