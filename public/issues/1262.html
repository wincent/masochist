Bug #1262: Wrong CSS class applied to [mailto:user@example.com email me]

Kind bug
Product wikitext
When Created 2009-03-26T19:49:06Z, updated 2009-03-27T07:12:02Z
Status closed
Reporter Greg Hurrell
Tags no tags

Description

In the example:

[mailto:win@wincent.com email]

This is a "mailto" URL and it ends up with the "external" CSS class being applied to it rather than the "mailto" class.

  2. Greg Hurrell 2009-03-26T20:20:06Z

    At the same time as fixing the class, probably makes sense to fix up the linked text as well.

    ie. Note how we given:

    mailto:user@example.com

    We produce:

    <a href="mailto:user@example.com">mailto:user@example.com</a>

    When it would perhaps be nicer if we produced:

    <a href="mailto:user@example.com">user@example.com</a>

    Another possibility is that we could also (or instead) start accepting input like:

    [user@example.com email me]

    Not sure if that is too lax and could lead to false positives, though.
