Feature request #1224: Add ability to tweak issue meta-data while submitting a comment

Kind feature request
Product wincent.com
When Created 2009-02-13T09:11:11Z, updated 2009-05-04T16:12:03Z
Status open
Reporter Greg Hurrell
Tags no tags

Description

I'm noticing that a common work-flow is to add a comment and then immediately after tweak some meta-data on an issue (for example, marking it as "closed" or something).

So perhaps it would be a good idea to provide a way of doing this in a single step?

    I often add an issue and then immediately change it from "new" to "open"; it might be good if admin users were offered a pop-up to set the issue status to "open" immediately when creating the issues.

    This is now done (see ticket #1250).

    As far as the UI for this is concerned, we might take a leaf out of Sifter's book. They offer radio buttons at the top of the comments box (obviously I'd want this for admins only):

    *--------------------------------------------------------*
|        [ ] new <-- [*] open -->  [ ] closed            |     
|--------------------------------------------------------|
|                                                        |
|                                                        |
|                                                        |
|                                                        |
|                                                        |
|                                                        |
*--------------------------------------------------------*

    Issue status is obviously the most common meta-data that one would want to manipulate while making a comment (ie. "marking this one as closed"). If the need arises I guess I could consider adding more fields, revealable using a disclosure triangle, but I am inclined to think that I probably won't need to do it that often. For the small number of times I do need to do it the standard AJAX updates are probably good enough.
