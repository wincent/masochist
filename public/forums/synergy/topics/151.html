  2. Greg Hurrell 2006-05-06T05:20:38Z

    I've made an entry in the feature requests database for this one:

    http://wincent.com/a/support/bugs/show_bug.cgi?id=420

    It is certainly possible to implement this feature in Synergy but because of the different comms engine it's not as straight forward to do it cleanly. The main difference is that Synergy Advance uses threaded communications so the user interface should remain responsive no matter what iTunes is doing, even if it blocks communication indefinitely; Synergy, on the other hand, does its comms with iTunes from the main thread and so is vulnerable to beach-balling if iTunes ever fails to respond.
