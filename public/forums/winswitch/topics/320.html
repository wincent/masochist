  2. Greg Hurrell 2005-01-01T18:37:37Z

    WinSwitch will show the root user as logged in if and only if both of the following two conditions are met:

    1. You've told it to display the root user in the preferences (*see note below).

    2. There is a process called "loginwindow" owned by user "root", which would indicate that the root user is logged in.

    You'll notice that it doesn't care with the root user is enabled, or if you've set a password for it. It only checks for the above two conditions. As for whether this is something to worry about, I don't know. The next time you see it, you should fire up Activity Monitor and see what's happening.

    The Apple FUS menu will never show the root user, whether its logged in or not.

    • Note: There is a minor bug in WinSwitch 3.0b where the preferences for displaying the root user are wired up back-to-front. So when "Show root user" is turned off, WinSwitch acts as if it's turned on and vice versa. Already fixed here, you'll see the fix in the next release.
