    Just an FYI, but in case CommandT doesn't seem to be responding when you type characters to filter the list of candidate files, it might be because you didn't install using the system ruby interpreter. I use rvm for my local account and forgot to switch to the system ruby while installing CommandT. A quick repeat of the install using the system ruby fixed this.
